## рҹ’Ў The Problem This Solves

Two hotel properties in Patna. One paper register. Zero visibility into occupancy, revenue, or guest history.

**StaySync** replaces that paper chaos with a production-grade Android app вҖ” offline-first, multi-property, role-protected вҖ” built entirely with Flutter and Appwrite from scratch in 8 iterative phases over 14 weeks.

This is **not a tutorial clone**. Every line of code was written to solve a real operational problem.

## вңЁ Features

| Module | What It Does |
|---|---|
| рҹ”җ **Auth + RBAC** | Email/password login via Appwrite with 3 role tiers вҖ” Admin, Manager, Staff. Session persisted locally. |
| рҹҸЁ **Hotel & Room Management** | Admin-only CRUD for properties and rooms. Room status auto-updates on booking events. |
| рҹ“Ӣ **Bookings + KYC** | Full guest form вҖ” DOB, ID proof photo, guest photo, double-booking prevention (client-side + Appwrite). |
| рҹ§ҫ **Guest Tab (POS)** | Per-booking charge tracking (beverages, meals, extras) with staff attribution. |
| рҹ’° **Smart Billing** | Auto-calculated checkout: `Total = (nights Г— room rate) + tab items`. |
| рҹ“Ҡ **Dashboard Analytics** | KPI cards, `fl_chart` monthly revenue graph, colour-coded room occupancy grid. |
| рҹ”„ **Offline-First Sync** | SQLite write queue вҖ” works without internet, syncs in background. Conflict resolution: *server-wins* for bookings, *client-wins* for items. |
| рҹҢҷ **Polish & UX** | Empty states, error states, form validation (regex), вӮ№ Indian currency, Dark mode вҖ” Material 3. |

в”Ӯ

в”Ӯ   в”ңв”Җв”Җ models/         # JSON serialisation / deserialisation

в”Ӯ   в”ңв”Җв”Җ entities/       # AppUser, Hotel, Room, Booking, Customer, BookingItem

    в”ңв”Җв”Җ auth/           # Login screen, AuthNotifier
    в”ңв”Җв”Җ hotels/         # Hotel list, form, room list, room form
    в”ңв”Җв”Җ bookings/       # Booking list, detail, form (KYC + imagepicker)

## рҹ§  Engineering Highlights

**Offline-First Architecture**
All reads hit SQLite first вҖ” zero network latency for the user. Writes go to the local queue immediately and sync to Appwrite in the background. If the network is down, operations never block.

**Conflict Resolution Strategy**
- Bookings: *server-wins* вҖ” the Appwrite record is the source of truth for check-in/check-out to prevent race conditions.
- Guest tab items: *client-wins* вҖ” staff additions are trusted locally since they happen in real-time at point of sale.

**RBAC Navigation Guards**
GoRouter's `redirect` callback checks the cached user role on every navigation. Attempting to access an admin-only route as Staff silently redirects to the role home вҖ” no flash, no error.

**KYC Photo Pipeline**
`image_picker` вҶ’ `flutter_image_compress` (80% quality) вҶ’ Appwrite Storage bucket вҶ’ URL stored in customer document. Works from both camera and gallery for emulator compatibility.
